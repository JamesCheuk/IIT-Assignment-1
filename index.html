Leadership Style

The media (movies, television shows) portrays as the ideal leader as a person who is front and centre of everything, who makes all the decisions when they need to be made while leading the group. A contrarian leader is the opposite of that. A contrarian leader is not afraid of delegating the decision making to someone else and are willing to hold off on making a decision that can be reasonably put off to another day. Regarding the first point, it may come across as being afraid to lead and take responsibility in a team. However, delegating the decision making does not mean delegating the responsibility too. A contrarian leader will assume responsibility for the decisions they delegate to others. This is an advantage as it allows other team members to showcase their qualities without fear, resulting in a higher quality of work and a more cohesive unit. The second point is a double-edged sword, as grasping the correct timing is more art than science. An advantage of waiting is that unforeseen options may open up for you, while at the same time waiting too long can result in options being lost. In a team environment, this particular trait may come in handy when dealing with conflict, as the extra time may allow the group to calm down and find a better solution to the conflict before making the final decision.

Project Idea

Location Triggered Alarm 'Never Miss a Stop'

Overview

This project will aim to create a mobile application which has a location triggered alarm. The user will be able to choose a location from a map and set how far away from the target location the alarm will go off. It will also contain features that will save the users’ alarms, allow certain customisation options for the alarms, offer online or offline option and operate in the background like regular stock alarms that come with all mobile devices.

Motivation

This would be an application that is targeted towards users who frequently travel on public transportation. With the busy and fast paced lifestyle of our everyday lives, moments on public transportation becomes a great time to catch up on some sleep. As a result of catching up on some sleep, we may inadvertently miss our stop and thus waste valuable time and this can cause anxiety and stress that would impact the quality of sleep. This application aims to put these anxieties aside so the user can sleep knowing this application will wake them up when they are nearing their destination.

Description

This application in the simplest terms is an alarm with location tracking capabilities. The application will require access to the device’s time and date, GPS services and limited internet access. In order to minimise data usage, a map of the city of the user’s choosing will be installed when the application is opened for the first time, which will allow offline usage.
The application will open to a home screen, which will be a map with the user’s current location displayed. At the top of the screen will be:
• A search bar where the user can find their destination and when they press search, they will be directed to the search result screen
• On the left side will be a menu that can be accessed by swiping the left edge of the screen that will have buttons for viewing existing saved alarms, updating the installed map and settings
The search result screen will have a display in the top half of the screen showing the destination, and the bottom half of the screen will have, with scrolling function:
• A drop box for the user to select how far away from the destination for the alarm to start going off (from 300m to 1km)
• A text box to give the alarm a name
• An option to select a ring tone from existing music files (it will default to the user’s current ring tone)
• A checkbox for whether it should be on repeat. If checked, new options will appear allowing the user to select which day(s) to repeat on and after what time to become active
• Two buttons side by side, one to save the alarm, one to cancel and return to the previous screen
The saved alarms screen will be where the user will be able to view all their saved alarms, with each save file displaying its name, destination, distance to destination to ring, whether it repeats and the option to turn the alarm on/off. The user will also be able to make modifications to existing saved alarms by tapping the save file which will lead the user to the search result screen to make the changes and save them. This screen will also offer the user the choice to delete any saved alarm by dragging it to a disposal area at the bottom of the screen.
The update maps button from the side menu will connect the device to a server that holds all the maps available to the users. A comparison will be made of the map version on the device against the version available on the server, if the versions do not match the user will be given a prompt to update the existing map. Prior to updating the map, if the device is not connected to Wi-Fi the user will be given a warning regarding excessive data usage.
The settings page will be where the user can choose their tracking method (GPS, network or both), view application version, patch notes and contact the developer should any issue arise while they are using the application.

Tools and Technologies

The tools and technologies that would be required to create this application include the Java programming language, an Integrated Development Environment (IDE) most likely to be Eclipse, a map database and a server to host the map database.
Java is an object-oriented programming language, which means the software is organised around data instead of functions and logic. This allows the developer to focus on what data they would need and how these data would interact with each other to produce the desired result. This would all be done on the Eclipse IDE which offers the developer various tools they would need to develop the application such as code editor, compiler and debugger.
The map database that the application will use will be limited to a select few large cities around the world where there is an extensive public transportation network available.

Skills Required

The skills that would be required for this project is outstanding knowledge of the Java programming language and how to use it to achieve your goals and being familiar with the Eclipse IDE so that development on it can progress smoothly. Some knowledge on how to set up and maintain a server would also be ideal, probably SQL (Structured Query Language).
Acquiring the skills listed above would not be difficult to achieve, as Java and SQL are very common and there are plenty of people who would be familiar with both. Learning both would also not pose a problem as there are many resources available that will teach someone how to use Java and SQL. As for the server, there are places where you can hire a server that suits your needs (e.g. hire-intelligence). The biggest problem may be finding the map database to be used alongside the application.

Outcome

Assuming the project is successful, it will allow regular public transportation users to rest on their journey without having to worry about oversleeping and missing their stop. If this application gets enough following, it would be possible that the stock alarms that come with most mobile devices will look to incorporate a location trigger function and become the new standard, and this project will be the spark that brings about this change.
